# Contributing to COMMS-UNITY

## The Idea

Everyone forks this. Everyone builds their own version. The best features get merged back. Eventually, different forks become different "rooms" that plug into each other — your version handles customer service, someone else's handles education, another handles legal aid. All connected.

## How It Works

1. **Fork the repo**
2. **Build your thing** — a feature, a whole new room, a skin, whatever
3. **Submit a PR** if you want it merged into the main platform
4. **Or keep your fork** — run your own instance, customize it for your community

## What We Need

- **Voice channels** — LiveKit integration, Xbox-style join/leave
- **Video/screen sharing** — teaching mode
- **Image/file uploads** — Supabase Storage integration
- **Real-time messaging** — replace polling with WebSockets
- **Builder marketplace** — search by skills, hire each other
- **Mobile app** — React Native or PWA
- **Skins/themes** — let communities customize their look
- **Room system** — different instances that can federate
- **AI integration** — ARAYA or any AI that can moderate, post, help

## Rules

1. **Builders only** — if you're here to break things, you're in the wrong place
2. **No gates** — don't add authentication barriers or paywalls to core features
3. **Keep it simple** — if a grandmother can't figure out your UI, simplify it
4. **MIT license** — everything stays open
